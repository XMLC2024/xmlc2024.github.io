如果你不想尝试这种《麻烦》的方法，你已经可以退出去了

Gridea

##如果您很有《实力》可以跳过此步骤
您可以点击此处打开Gridea - 官方网站，但是这是非开源的Gridea版本，请点击“开源客户端”下载免费版本。或Windows用户点击此处直接下载Gridea - Windows

你也可以点击这里，打开GITHUB上的Gridea
Gridea - GITHUB

GITHUB PAGES

开始前，请点击Github - 网址
如果无法正常连接，请往下看

Github 无法正常连接

这是非常正常的一种情况，有两种解决办法

  1. Magic
  2. 使用工具Watt Toolkit的Github加速解决此问题，如果你不会，请自行询问一个叫百度的东西,或者直接跳转到百度

回归主题

如果你成功进去了GITHUB,请您先注册一个账号，如果你也不会，麻烦测试一下英语并询问百度如何注册

注册成功后,请:

  1. 点击右上角的加号
  2. 选择new repository创建新的存储库
  3. 填写Repository name存储库名称，为“你的用户名+github.io"
    例子:example.github.io
  4. 选择Public
  5. 点击绿色小按钮

如果你成功进入了界面，那么你已经成功1/4了!

GITHUB APP

接下来，还有东西正在等着你！
点击此链接[创建GITHUB APP]来创建一个GITHUB APP。
啥也不用动，直接确认即可。
点击[Github APP主页]，找到你刚刚创建的APP。
将Client ID与Client secrets复制好。

注意：Client secrets一旦丢失，没有后来！
如果丢失，你只能重开，一定保存好!

配置Gridea

打开Gridea,选择远程。
基础配置:

  1. 平台选择Github Pages,域名填写:你的名字+.github.io

不含https://

  1. 仓库名称同你的名字+.github.io
  2. 分支为master不用动，默认的
  3. 仓库用户名就是你的用户名
  4. 令牌(Token):Github TOKEN,Note随便写，勾选repo所有内容，划到最底部，生成即可。
  5. 检测远程连接

如果显示成功,您可以点击保存并继续
如果失败，请你检查你的操作，如果仍有问题，划到最底部进行留言。

到此为止，最基础的已经配置完毕。
